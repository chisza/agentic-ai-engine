# Computer Science and Software Engineering Glossary

## Networking and Protocols

**TCP/IP (Transmission Control Protocol/Internet Protocol)**: The foundational communication protocols of the internet. TCP ensures reliable, ordered delivery of data between applications; IP handles addressing and routing of packets across networks.

**REST (Representational State Transfer)**: An architectural style for distributed hypermedia systems. RESTful APIs use HTTP methods (GET, POST, PUT, DELETE) and treat resources as URLs, enabling stateless communication between clients and servers.

**HTTP/HTTPS (HyperText Transfer Protocol / Secure)**: The protocol used for transmitting web pages. HTTPS adds TLS encryption for secure data transfer.

**DNS (Domain Name System)**: A hierarchical naming system that translates human-readable domain names (e.g. example.com) into IP addresses.

**API (Application Programming Interface)**: A set of rules and protocols that allows different software applications to communicate with each other.

## Software Architecture

**Microservices**: An architectural style where an application is built as a collection of small, independently deployable services, each running in its own process and communicating via APIs.

**Monolith**: A software application built as a single, unified unit where all components are tightly coupled and deployed together.

**Container**: A lightweight, portable software package that includes everything needed to run an application: code, runtime, libraries, and settings. Docker is the most common containerisation platform.

**Kubernetes (K8s)**: An open-source container orchestration system for automating deployment, scaling, and management of containerised applications.

**CI/CD (Continuous Integration/Continuous Delivery)**: A set of practices that automate the building, testing, and deployment of software. CI merges code frequently; CD automates the release pipeline.

## Data and Databases

**SQL (Structured Query Language)**: A domain-specific language for managing and querying relational databases.

**NoSQL**: A category of database management systems that store data in formats other than relational tables (e.g. document stores, key-value stores, graph databases).

**ORM (Object-Relational Mapping)**: A technique that lets you query and manipulate a relational database using an object-oriented paradigm in your programming language.

**ACID (Atomicity, Consistency, Isolation, Durability)**: Properties that guarantee reliable database transactions.

**Index**: A data structure that improves the speed of data retrieval operations in a database.

## Programming Concepts

**Algorithm**: A finite sequence of well-defined instructions to solve a problem or accomplish a computation.

**Data Structure**: A specialised format for organising, processing, retrieving, and storing data (e.g. array, linked list, tree, hash map).

**Object-Oriented Programming (OOP)**: A programming paradigm based on "objects" that contain both data (attributes) and behaviour (methods). Core principles: encapsulation, inheritance, polymorphism, abstraction.

**Functional Programming**: A programming paradigm that treats computation as the evaluation of mathematical functions, avoiding mutable state and side effects.

**Recursion**: A method where the solution to a problem depends on solutions to smaller instances of the same problem.

**Big O Notation**: A mathematical notation used to describe the limiting behaviour of a function, typically used to classify algorithm efficiency by time or space complexity.

## Machine Learning and AI

**Machine Learning (ML)**: A subset of AI that enables systems to learn and improve from experience without being explicitly programmed.

**Neural Network**: A computing system inspired by the human brain, consisting of layers of interconnected nodes (neurons) that process information.

**LLM (Large Language Model)**: A type of AI model trained on vast amounts of text data, capable of generating, summarising, translating, and reasoning about text.

**Training**: The process of feeding data to a machine learning model so it can learn patterns and improve its predictions.

**Inference**: Using a trained model to make predictions or generate output for new, unseen input.

**RAG (Retrieval-Augmented Generation)**: A technique that combines a retrieval system (fetching relevant documents) with a generative model to ground responses in factual, up-to-date information.

## Security

**Authentication**: The process of verifying the identity of a user, device, or system.

**Authorisation**: The process of determining what resources or actions an authenticated entity is permitted to access.

**Encryption**: The process of encoding data so that only authorised parties can decode and read it.

**TLS/SSL (Transport Layer Security / Secure Sockets Layer)**: Cryptographic protocols that provide secure communication over a network.

**OAuth 2.0**: An open standard for access delegation, allowing users to grant third-party apps limited access to their resources without sharing passwords.
